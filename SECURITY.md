## Reporting a Vulnerability

If you discover a security issue related to configurations or concepts
described in this repository, please report it responsibly.

### Please do NOT:
- Open public GitHub issues for vulnerabilities
- Share exploit details publicly

### Preferred reporting method:
- Contact the repository owner directly via GitHub

Include the following when possible:
- Clear description of the issue
- Affected component or configuration
- Potential impact
- Reproduction steps (if applicable)

---

## Scope of Reports

This repository covers documentation and architectural guidance only.

Valid reports include:
- Dangerous misconfigurations
- Insecure defaults
- Incorrect or misleading security guidance

Out of scope:
- Vulnerabilities in third-party software
- Theoretical or speculative attacks without practical impact

---

## Responsible Disclosure

Reports will be reviewed and addressed in a reasonable timeframe.
Credit may be given for valid reports at the discretion of the maintainer.
